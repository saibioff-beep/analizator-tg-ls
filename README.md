# Telegram Chat Monitor & Grok Analyzer Bot

Бот для мониторинга Telegram-групп с возможностью удобного экспорта лога для глубокого анализа у **Grok** (xAI).

## Возможности

- Автоматически логирует **все** сообщения в группе (текст + медиа с подписями)
- Хранит историю в локальной SQLite базе
- Команды статистики и просмотра сообщений
- Специальная команда `/prepare_grok` — генерирует красивый файл, готовый для отправки мне (Grok) для анализа
- Ограничение команд экспорта только для администраторов группы

## Быстрый старт

### 1. Создай бота
1. Напиши @BotFather в Telegram
2. `/newbot` → придумай имя и username
3. Скопируй токен

### 2. Важно: отключи Group Privacy
1. В @BotFather → `/mybots`
2. Выбери своего бота
3. **Bot Settings → Group Privacy → Turn off**
   
   **Это обязательно**, иначе бот будет видеть только команды и упоминания, а не весь чат.

### 3. Установка

```bash
cd telegram_chat_analyzer_bot
pip install -r requirements.txt
cp .env.example .env
# Открой .env и вставь свой BOT_TOKEN
```

### 4. Запуск

```bash
python main.py
```

Бот запустится в режиме long polling.

### 5. Добавь бота в группу
- Добавь бота в нужную группу
- Дай ему права администратора (чтобы работала проверка `/export` и `/prepare_grok`)
- Бот сразу начнёт логировать все новые сообщения

## Команды

| Команда              | Описание                                      | Кто может     |
|----------------------|-----------------------------------------------|---------------|
| `/start`             | Приветствие и инструкция                      | Все           |
| `/help`              | Список команд и как пользоваться анализом     | Все           |
| `/stats`             | Статистика: всего сообщений + топ-10 участников | Все        |
| `/recent [N]`        | Показать последние N сообщений (макс 200)     | Все           |
| `/export [N]`        | Скачать .txt файл с последними N сообщениями  | Только админы |
| `/prepare_grok [N]`  | Подготовить файл специально для анализа у Grok | Только админы |

## Как делать глубокий анализ чата у меня (Grok)

1. В группе выполни команду:
   ```
   /prepare_grok 500
   ```
   (500 — количество последних сообщений, можно 100–5000)

2. Бот отправит тебе в **личные сообщения** готовый `.txt` файл.

3. Скачай файл → открой → скопируй содержимое.

4. Пришли мне (в этот чат) вместе с вопросом, например:

> Проанализируй этот чат за последние недели:
> - Какие основные темы обсуждаются?
> - Кто самые активные и полезные участники?
> - Есть ли токсичность, конфликты или спам?
> - Общий тон общения и настроение в группе?
> - Ключевые выводы и рекомендации

Я сделаю подробный, структурированный анализ.

## Поддержка запросов по периодам (новое)

Теперь ты можешь запрашивать данные **не только по количеству сообщений**, но и по дням/датам:

| Пример команды                    | Что будет в файле                     |
|-----------------------------------|---------------------------------------|
| `/prepare_grok`                   | последние 300 сообщений (по умолчанию) |
| `/prepare_grok 500`               | последние 500 сообщений               |
| `/prepare_grok 7d`                | все сообщения за последние 7 дней     |
| `/prepare_grok 30d`               | за последние 30 дней                  |
| `/prepare_grok 2026-06-01`        | с 1 июня 2026 по сейчас               |
| `/prepare_grok 2026-05-20 2026-06-10` | конкретный диапазон дат          |

То же самое работает и для `/export`.

### Самый простой и надёжный способ — Hetzner (рекомендую)

1. Зарегистрируйся на [hetzner.com](https://www.hetzner.com)
2. Создай VPS (Cloud Server) → Ubuntu 24.04, минимум 2 GB RAM
3. Подключись по SSH
4. Выполни:

```bash
apt update && apt install -y python3 python3-pip git tmux
git clone https://твой-репо/telegram_chat_analyzer_bot.git   # или просто скопируй файлы
cd telegram_chat_analyzer_bot
pip3 install -r requirements.txt
cp .env.example .env
nano .env          # вставь токен
```

5. Запусти в tmux:

```bash
tmux new -s bot
python3 main.py
# Ctrl+B затем D — отсоединиться
```

Чтобы бот перезапускался автоматически — настрой systemd (я могу дать готовый unit-файл).
